任务：实现团子赛跑游戏的高次数蒙特卡洛模拟，并导出结果文档

请根据以下游戏规则，编写一个可运行的模拟程序，用于计算每个普通团子的：

- 获胜概率
- 获胜期望
- 平均名次
- 各名次概率分布
- 99% 置信区间
- 模拟结果文档

要求将模拟次数扩展至极高，至少为 1,000,000 次以上。

请优先保证规则实现正确，其次再进行性能优化。

------------------------------------------------------------
一、总体要求
------------------------------------------------------------

请实现一个完整的模拟程序，建议使用 Python。

程序需要支持：

python simulate.py --trials 1000000 --seed 42 --output results.csv

其中：

- --trials：模拟次数，默认至少 1,000,000
- --seed：随机种子，用于复现实验
- --output：导出模拟结果文件路径

最终至少生成：

1. simulate.py
   - 可直接运行的模拟程序
2. results.csv
   - 模拟统计结果
3. 可选：README.md
   - 说明规则假设、运行方式、输出字段含义

如果规则中存在歧义，请不要中断实现，而是在代码注释和 README 中明确写出采用的假设。

------------------------------------------------------------
二、游戏地图
------------------------------------------------------------

游戏共有 32 个点位，编号为：

0 - 31

地图为环形终点制，普通团子从 0 号位向 31 号位前进。

特殊点位如下：

GREEN_TILES = {2, 10, 15, 22}
RED_TILES = {9, 27}
BLACK_TILES = {5, 19}

1. 绿色点位

当团子的移动终点为绿色点位时：

- 将该团子向前移动 1 格
- 如果该团子带着上方堆叠团子移动，则整个移动组一起向前移动 1 格
- 如果该团子被其他团子堆叠在上方，也按正常堆叠移动规则处理

普通团子向前方向：

0 -> 1 -> 2 -> ... -> 31

哈基布向前方向：

31 -> 30 -> 29 -> ... -> 0

2. 红色点位

当团子的移动终点为红色点位时：

- 将该团子向后移动 1 格
- 如果该团子带着上方堆叠团子移动，则整个移动组一起向后移动 1 格

普通团子向后方向：

31 -> 30 -> ... -> 0

哈基布向后方向：

0 -> 1 -> ... -> 31

3. 黑色点位

当普通团子的移动终点为黑色点位时：

- 随机重新堆叠该点位上的所有普通团子
- 不包括特殊团子“哈基布”
- 哈基布不参与黑色点位重排

普通团子如下：

CUBIES = [

SPECIAL = "哈基布"

哈基布不参与最终结算排名。

------------------------------------------------------------
四、初始状态
------------------------------------------------------------

游戏开始时：

1. 所有普通团子位于 0 号点位
2. 所有普通团子以随机顺序堆叠在 0 号位
3. 堆叠顺序采用：bottom -> top

例如：

["千咲", "莫宁", "琳奈"]

表示：

- 千咲在最底部
- 琳奈在最顶部

4. 越晚堆叠的团子排名越靠前，即越靠近顶部，排名越靠前
5. 哈基布游戏开始时位于 31 号点位
6. 哈基布不参与最终排名结算

------------------------------------------------------------
五、基础回合规则
------------------------------------------------------------

游戏按回合进行。

每一回合开始时：

1. 随机排列普通团子的行动顺序
2. 每个普通团子依次行动
3. 普通团子默认骰子为 1~3，等概率随机
4. 某些团子的技能会改变骰子结果或移动距离
5. 从游戏开始第三回合后，哈基布开始行动
   - 建议解释为：第 4 回合开始行动
   - 若采用其他解释，请在注释中注明

------------------------------------------------------------
六、普通团子移动规则
------------------------------------------------------------

普通团子移动时：

1. 找到该团子当前所在点位的堆叠
2. 如果该团子上方还有其他普通团子，则该团子会带动自己以及上方所有团子一起移动
3. 如果该团子位于堆叠顶部，则只移动自己
4. 下方团子不会被带动
5. 移动组到达目标点位后，放置在目标点位现有堆叠的最上方
6. 移动组内部原有上下顺序保持不变

示例：

某点位当前堆叠 bottom -> top:

[A, B, C]

如果 B 行动，则移动组为：

[B, C]

A 留在原点位。
B 和 C 一起移动到目标点位。

------------------------------------------------------------
七、游戏结束条件
------------------------------------------------------------

当任意普通团子移动至 31 号位时，游戏立即结算。

结算规则：

1. 先越过 31 号位的团子为第一名
2. 如果在越过 31 号位时存在堆叠，则按照堆叠顺序从上到下排列名次
3. 其余普通团子按照当前位置排序
4. 越靠近 31 号位，排名越靠前
5. 同一格内，越靠上的团子排名越靠前
6. 哈基布不参与最终排名

建议实现：

- 为每个普通团子维护当前位置 position
- 普通团子位置限制在 0~31
- 普通团子前进时不能超过终点，若超过则视为到达 31
- 只要有普通团子位置达到 31，立即结束并结算

------------------------------------------------------------
八、特殊团子：哈基布
------------------------------------------------------------

哈基布规则如下：

1. 哈基布游戏开始时位于 31 号点位
2. 哈基布不参与最终结算
3. 地图所有特殊效果对哈基布生效
4. 游戏开始第三回合后，哈基布开始行动
   - 建议从第 4 回合开始行动
5. 哈基布按照 31 -> 0 的方向移动
6. 哈基布每轮投掷 1~6 点，等概率
7. 如果哈基布单次移动终点遇到普通团子：
   - 哈基布永远堆叠于该点位最底部
   - 普通团子的堆叠顺序不改变
8. 如果在整轮行动中，正向前进的普通团子点位大于哈基布所在点位：
   - 哈基布传送回 31 号位重新开始
9. 哈基布不参与黑色点位普通团子的随机重排
10. 普通团子移动时，默认不会带动哈基布移动
11. 哈基布在移动时，会带动上方团子一起移动

建议实现方式：

- 哈基布独立记录位置 haji_position
- 不将哈基布放入普通团子的 stack 列表
- 当哈基布与普通团子同格时，只视为该格最底部的特殊存在
- 普通团子排名与堆叠判断不考虑哈基布
- 每轮结束时检查是否所有普通团子位置是否大于哈基布位置
  - 若是，则哈基布回到 31

------------------------------------------------------------
九、团子技能规则
------------------------------------------------------------

以下技能需要完整实现。

1. 千咲

技能描述：

掷骰后，若点数是本轮所有团子中最小的之一，额外前进 2 步。

实现要求：

- 每轮开始时，需要先为本轮所有普通团子生成基础骰子点数
- 千咲的基础骰子范围为 1~3
- 找出本轮所有普通团子的基础骰子最小值
- 如果千咲本轮基础骰子等于该最小值，则千咲本次移动距离额外 +2
- 如果存在多个团子并列最小，千咲仍然满足条件
- 注意：这里比较的是基础骰子点数，不是技能修正后的最终移动距离

建议流程：

每轮开始：
1. 为所有普通团子生成本轮基础骰子
2. 判断本轮最小骰子点数
3. 若千咲基础骰子 == 最小值，则千咲移动距离 = 基础骰子 + 2

2. 莫宁

技能描述：

骰子固定循环：3 -> 2 -> 1，每回合依次推进循环。

实现要求：

- 莫宁不随机掷骰
- 莫宁的基础骰子按照以下顺序循环：

第 1 次行动：3
第 2 次行动：2
第 3 次行动：1
第 4 次行动：3
第 5 次行动：2
第 6 次行动：1
...

- 每当莫宁实际行动一次，循环推进一次
- 如果因为某种规则导致莫宁未行动，则是否推进循环需要在注释中说明
- 当前规则中莫宁一般都会行动

3. 琳奈

技能描述：

每回合 60% 概率按双倍点数前进，20% 概率本回合完全无法移动，剩余 20% 正常移动。

实现要求：

- 琳奈基础骰子为 1~3
- 每次琳奈行动时，额外判定技能：
  - 60%：最终移动距离 = 基础骰子 * 2
  - 20%：最终移动距离 = 0，本回合不移动
  - 20%：最终移动距离 = 基础骰子
- 如果最终移动距离为 0：
  - 不移动
  - 不触发地图点位效果
  - 不改变堆叠

4. 爱弥斯

技能描述：

全场比赛仅限使用一次。
自身经过中点（15号位）后，若前方存在非哈基布团子，则传送至最近团子的最顶部。

注意：原文中“非哈基布团子”，即普通团子。

实现要求：

- 爱弥斯技能每场比赛最多触发一次
- 当爱弥斯首次经过或到达 15 号位后，检查其前方是否存在其他普通团子
- “前方”指位置大于爱弥斯当前位置，且尚未到达终点外的普通团子
- 找到前方距离最近的普通团子所在点位
- 将爱弥斯从当前堆叠中移除
- 将爱弥斯放到该最近普通团子所在点位的最顶部
- 被传送点位的原有堆叠顺序保持不变
- 爱弥斯传送后技能标记为已使用
- 如果经过中点时前方没有普通团子：
  - 建议技能仍视为已触发但无效果，或者保留到下次满足条件
  - 请在代码注释中明确采用哪种假设

推荐假设：

爱弥斯经过中点后第一次进行技能检查。
如果前方存在普通团子，则传送并消耗技能。
如果前方不存在普通团子，则不消耗技能，之后仍可再次检查。

5. 守岸人

技能描述：

骰子只会出现 2 或 3，各 50% 概率。

实现要求：

- 守岸人的基础骰子不是 1~3
- 守岸人每次行动基础骰子为：
  - 2，概率 50%
  - 3，概率 50%

6. 柯莱塔

技能描述：

28% 概率以掷出的点数双倍前进，其余正常移动。

实现要求：

- 柯莱塔基础骰子为 1~3
- 每次行动时进行技能判定：
  - 28%：最终移动距离 = 基础骰子 * 2
  - 72%：最终移动距离 = 基础骰子

------------------------------------------------------------
十、行动顺序与骰子生成建议
------------------------------------------------------------

由于千咲的技能依赖“本轮所有团子中最小的骰子点数”，建议每轮流程如下：

1. round_number += 1
2. 随机排列普通团子行动顺序
3. 为本轮所有普通团子预生成基础骰子点数
   - 千咲：1~3
   - 莫宁：固定循环 3,2,1
   - 琳奈：1~3
   - 爱弥斯：1~3
   - 守岸人：2 或 3
   - 柯莱塔：1~3
4. 计算本轮普通团子的最小基础骰子点数
5. 按行动顺序依次行动普通团子
6. 每个团子根据技能修正最终移动距离
7. 执行移动
8. 移动后触发地图特殊点位
9. 检查是否到达 31 号位
10. 若到达，立即结算
11. 本轮普通团子行动结束后，如果回合数 >= 4，则哈基布行动
12. 哈基布行动后触发地图特殊点位
13. 回合结束时检查哈基布是否需要传送回 31

如果你认为哈基布应插入随机行动顺序中，也可以采用该方式，但需要在注释中说明。

推荐采用：

哈基布从第 4 回合开始，在每轮所有普通团子行动结束后单独行动一次。

原因是原规则中描述为“游戏开始第三回合后，哈基布将按照 31->0 的顺序进行移动”，未明确说明其加入普通团子的随机行动序列。

排名优先级：

1. 位置越大，排名越靠前
2. 如果在同一位置，则堆叠越靠上，排名越靠前

示例：

bottom -> top
[A, B, C]

排名顺序：
C > B > A

最终排名：

第 1 名：最靠近或到达 31 的团子
第 2 名：次靠前的团子
...
第 6 名：最后的团子

99% 置信区间：

可使用二项分布正态近似：

p_hat = wins / n
se = sqrt(p_hat * (1 - p_hat) / n)
z = 2.5758293035489004
ci_low = p_hat - z * se
ci_high = p_hat + z * se

其中：

n = 总模拟次数

请将置信区间限制在 [0, 1] 范围内。

如果使用 Wilson 区间，也可以，但需要在输出中说明。
