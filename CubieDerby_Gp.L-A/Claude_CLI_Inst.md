# Claude 工作指南：CLI 版团子竞速模拟

> 用途：将本指南复制给 Claude，让其编写一个只在 CLI / 终端中显示结果的 Python 蒙特卡洛模拟程序。  
> 目标：不生成 CSV、Excel、图片、图表、PDF 或其他文件，只在终端中打印模拟结果。

---

## 1. 任务目标

请根据团子竞速游戏规则，编写一个 **Python CLI 程序**，通过至少 **1,000,000 次蒙特卡洛模拟**，计算每个普通团子的：

1. 胜率；
2. 各名次概率；
3. 平均名次；
4. 胜率的 99% 置信区间；
5. 平均名次的 99% 置信区间。

**结果只需要显示在 CLI / 终端中，不需要生成 CSV、Excel、图片、图表、PDF 或任何其他文件。**

---

## 2. CLI 程序要求

### 2.1 只打印，不写文件

程序只能使用 `print()` 或类似方式在终端输出结果。

不要生成以下文件：

```text
.csv
.xlsx
.png
.pdf
.json
.txt
```

不要使用类似以下写文件操作：

```python
to_csv()
savefig()
open(..., "w")
```

除非用户之后明确要求导出文件。

### 2.2 支持命令行参数

程序应支持以下运行方式：

```bash
python simulate_dango.py --n 1000000 --seed 20260514
```

参数说明：

```text
--n      模拟次数，默认 1000000
--seed   随机种子，默认 20260514
```

也应支持例如：

```bash
python simulate_dango.py --n 5000000 --seed 42
```

### 2.3 输出格式

使用纯文本表格即可，不依赖复杂第三方库。

可以使用 Python 格式化字符串，例如：

```python
print(f"{name:<8}{win_rate:>12.6f}{ci_text:>24}{avg_rank:>12.4f}{rank_ci_text:>24}")
```

不要求使用 `rich`、`tabulate` 等库，避免环境依赖。

## 3. 游戏地图规则

游戏地图共有 **32 个点位**，编号为 `0-31`，呈环形排列。

特殊点位如下：

```text
绿色点位：2、10、15、22
红色点位：9、27
黑色点位：5、19
```

### 3.1 绿色点位

移动终点为绿色点位的所有团子，将其向前移动 1 格，包括堆叠与被堆叠团子。

### 3.2 红色点位

移动终点为红色点位的所有团子，将其向后移动 1 格，包括堆叠与被堆叠团子。

### 3.3 黑色点位

当团子的终点为黑色点位时，随机重新堆叠该点位上的所有普通团子。

注意：

- 黑色点位随机重排不包含特殊团子哈基布；
- 若多个普通团子处于同一格，应维护严格的堆叠顺序。

## 4. 团子列表

普通团子包括：

```text
菲比
陆赫斯
玲奈
莫宁
弗诺诺
长离
```

特殊团子：

```text
哈基布
```

哈基布不参与最终排名结算。

## 5. 初始状态

游戏开始时：

1. 所有普通团子均位于 `0` 号点；
2. 所有普通团子以随机顺序堆叠；
3. 越晚堆叠的团子排名越靠前；
4. 哈基布初始位于 `31` 号点；
5. 哈基布不参与最终排名。

建议在代码中统一约定：

```text
列表索引 0 表示最底部；
列表最后一个元素表示最顶部。
```

例如：

```python
board[0] = ["弗诺诺", "玲奈", "菲比"]
```

表示：

```text
弗诺诺在最底部
玲奈在中间
菲比在最顶部
```

---

## 6. 每轮行动流程

### 6.1 行动顺序

每轮开始时，随机排列本轮参与行动的团子。

普通团子从游戏开始即参与行动。

哈基布在游戏开始第三回合后加入随机投掷序列。

默认实现假设：

```text
第 1、2、3 回合：哈基布不行动
第 4 回合起：哈基布加入行动序列
```

### 6.2 普通团子移动

普通团子默认随机前进 `1-3` 格。

如果行动团子处于堆叠中：

- 该团子会带动自己以及位于自己上方的所有团子一起移动；
- 位于该团子下方的团子不会移动。

移动后的团子组到达目标点位时，默认放置在目标点位已有普通团子的上方。

---

## 7. 终局条件与排名结算

游戏结束条件：

```text
当普通团子已经过 0 号点位并移动至 31 号位时，游戏结算。
```

建议每个普通团子维护：

```text
position: 当前地图点位，范围 0-31
distance: 累计正向移动距离
```

终点判定建议为：

```python
distance >= 31 and position == 31
```

结算规则：

1. 先抵达 `31` 号位的团子为第一名；
2. 若抵达 `31` 号位时存在堆叠，则按照堆叠顺序从上至下排名；
3. 未抵达终点的团子按累计前进距离从高到低补全排名；
4. 若累计距离相同，则按当前堆叠顺序从上到下排序。

---

## 8. 哈基布规则

哈基布规则如下：

1. 游戏开始时位于 `31` 号点；
2. 不参与最终排名结算；
3. 地图所有特殊效果都对哈基布生效；
4. 游戏开始第三回合后加入随机投掷序列；
5. 哈基布按照 `31 -> 0` 的反方向移动；
6. 每轮可投掷 `1-6` 点；
7. 若哈基布单次前进终点遇到正向行进的普通团子，哈基布永远堆叠于最底部；
8. 若在整轮行动中，正向前进的普通团子点位大于哈基布所在点位，哈基布传送回 `31` 号位重新开始。

哈基布移动方向：

```python
hakibu_position = (hakibu_position - steps) % 32
```

当哈基布移动终点存在普通团子：

- 哈基布应放在该点位堆叠最底部；
- 哈基布不改变普通团子的相对顺序；
- 哈基布不参与黑色点位普通团子的随机重排。

每轮结束时检查：

```python
if any(normal_position > hakibu_position for normal_position in normal_positions):
    hakibu_position = 31
```

注意：这里比较的是地图点位编号，而不是累计距离。

## 9. 各团子技能实现

### 9.1 菲比

技能：

```text
50% 概率前进 1 格
```

默认解释：

- 菲比正常掷骰后，有 50% 概率额外前进 1 格；
- 额外移动应携带其上方堆叠团子；
- 额外移动后的终点也应触发地图点位效果。

### 9.2 陆赫斯

技能：

```text
触发绿色点位时可额外前进 3 格，但触发红色点位时额外后退 1 格
```

默认解释：

- 陆赫斯触发绿色点位时，在绿色点位基础前进 1 格之外，额外前进 3 格；
- 陆赫斯触发红色点位时，在红色点位基础后退 1 格之外，额外后退 1 格；
- 技能导致的额外移动默认不递归触发新的特殊格。

### 9.3 玲奈

技能：

```text
每回合中有 60% 概率按照双倍点数移动，但有 20% 概率当回合无法移动
```

默认判定顺序：

1. 先判定 20% 无法移动；
2. 若未无法移动，再判定 60% 双倍点数；
3. 若双倍触发，则本次骰子点数乘以 2。

伪代码：

```python
roll = randint(1, 3)

if random.random() < 0.2:
    actual_move = 0
else:
    if random.random() < 0.6:
        actual_move = roll * 2
    else:
        actual_move = roll
```

无法移动时，不触发地图点位效果。

### 9.4 莫宁

技能：

```text
骰子固定循环：3 -> 2 -> 1
```

实现要求：

- 莫宁不使用随机 `1-3` 骰子；
- 每次行动依次使用 `3, 2, 1, 3, 2, 1...`；
- 循环状态在单局游戏内持续维护；
- 每局游戏开始时重置为 `3`。

### 9.5 弗诺诺

技能：

```text
回合开始时，若该团子处于堆叠最底层，则在移动时额外前进 3 格
```

实现要求：

- 判断时点为“回合开始时”；
- 每轮开始记录弗诺诺是否处于所在堆叠的最底层；
- 若是，则本轮弗诺诺行动时额外前进 3 格；
- 即使行动前堆叠状态改变，也使用回合开始时的判定结果。

### 9.6 长离

技能：

```text
如果下方堆叠其他团子，下一个回合有 65% 的概率最后一个行动
```

实现要求：

- 若长离下方存在其他普通团子，则下一回合生成行动顺序时，有 65% 概率将长离移动到行动序列最后；
- 如果哈基布也在行动序列中，“最后一个行动”表示在所有参与行动者中最后行动。

## 10. 特殊格触发顺序

默认触发顺序如下：

1. 团子根据骰子与技能移动；
2. 到达移动终点；
3. 判断该点是否为绿色、红色或黑色；
4. 若为绿色，整体前进 1 格；
5. 若为红色，整体后退 1 格；
6. 若为黑色，随机重排该格普通团子；
7. 特殊格效果不递归触发；
8. 检查是否达到终局。

必须在代码注释或 CLI 输出中明确说明：

```text
本模拟默认特殊格效果不递归触发。
```

## 12. 99% 置信区间计算

### 12.1 胜率的 99% 置信区间

建议使用 Wilson 区间。

```python
z = 2.5758293035489004
p_hat = wins / N

denom = 1 + z**2 / N
center = (p_hat + z**2 / (2*N)) / denom
half_width = z * sqrt((p_hat*(1-p_hat)/N + z**2/(4*N**2))) / denom

lower = center - half_width
upper = center + half_width
```

并截断到 `[0, 1]`。

### 12.2 平均名次的 99% 置信区间

使用样本均值正态近似：

```python
mean_rank = np.mean(ranks)
std_rank = np.std(ranks, ddof=1)
se = std_rank / sqrt(N)

lower = mean_rank - z * se
upper = mean_rank + z * se
```

如果不使用 NumPy，可以用累计和与累计平方和计算均值与标准差。
