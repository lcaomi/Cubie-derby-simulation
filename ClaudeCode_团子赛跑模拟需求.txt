任务：基于规则实现团子赛跑游戏的蒙特卡洛模拟

请根据以下规则，实现一个可运行的模拟程序，用于计算每个普通团子的获胜期望与获胜概率。

要求：

1. 使用蒙特卡洛模拟方法。
2. 模拟次数至少为 1,000,000 次。
3. 对每个普通团子的胜率计算 99% 置信区间。
4. 输出每个普通团子的：
   - 第一名概率
   - 平均名次 / 期望名次
   - 99% 置信区间
   - 可选：排名分布，例如第 1、2、3、4、5、6 名概率
5. 特殊团子“哈基布”不参与游戏结算，但会影响比赛过程。
6. 请尽量保证规则实现清晰、可验证，并在代码中加入必要注释。
7. 如规则存在歧义，请先在代码中用注释标明你的假设，并保持实现一致。

------------------------------------------------------------
游戏地图
------------------------------------------------------------

游戏共有 32 个点位，编号为 0 到 31，呈环形排列。

普通团子从 0 号点位出发，正向移动，移动方向为：

0 -> 1 -> 2 -> ... -> 30 -> 31 -> 0 -> ...

特殊团子“哈基布”从 31 号点位出发，逆向移动，移动方向为：

31 -> 30 -> 29 -> ... -> 1 -> 0 -> 31 -> ...

地图中的特殊点位如下：

绿色点位：2、10、15、22
红色点位：9、27
黑色点位：5、19

特殊点位效果：

绿色点位：
当普通团子或哈基布的移动终点为绿色点位时，将其向前移动 1 格。

- 对普通团子来说，向前是正向移动。
- 对哈基布来说，向前是逆向移动。
- 如果该团子携带了堆叠在其上方的团子，则这些团子也一起移动。
- 如果该团子本身被其他团子压在下面，也需要按堆叠移动规则处理。

红色点位：
当普通团子或哈基布的移动终点为红色点位时，将其向后移动 1 格。

- 对普通团子来说，向后是逆向移动。
- 对哈基布来说，向后是正向移动。
- 如果该团子携带了堆叠在其上方的团子，则这些团子也一起移动。
- 如果该团子本身被其他团子压在下面，也需要按堆叠移动规则处理。

黑色点位：
当普通团子的移动终点为黑色点位时，随机重新堆叠该点位上的所有普通团子。

注意：

- 黑色点位重排不包括特殊团子“哈基布”。
- 如果哈基布也在该点位，则不参与随机重排。
- 黑色点位只影响该点位上的普通团子堆叠顺序。

团子与初始状态
------------------------------------------------------------

普通团子列表如下：

奥古斯塔
尤诺
弗诺诺
长离
今汐
卡卡罗

特殊团子：

哈基布

游戏开始时：

1. 所有普通团子均位于 0 号点位。
2. 普通团子在 0 号点位随机堆叠。
3. 堆叠顺序随机决定。
4. 越晚堆叠的团子排名越靠前，即位于堆叠更上方。
5. 哈基布位于 31 号点位。
6. 哈基布不参与最终排名结算。

建议用如下方式表示堆叠：

bottom -> top

例如：

["卡卡罗", "尤诺", "今汐"]

表示：

- 卡卡罗在最底部
- 今汐在最顶部

排名判断时，同一格内越靠上的团子排名越靠前。

------------------------------------------------------------
基础移动规则
------------------------------------------------------------

游戏按“回合”进行。

每一回合开始时：

1. 随机排列所有当前参与行动的普通团子的行动顺序。
2. 从第 3 回合之后，哈基布加入随机投掷序列。
3. 每个普通团子每次行动投掷 1~3 点，等概率。
4. 哈基布每次行动投掷 1~6 点，等概率，并沿逆向方向移动。
5. 按行动顺序依次执行移动。

普通团子移动时：

- 如果移动的团子上方还有其他普通团子，则该团子会带动自己以及上方所有团子一起移动。
- 如果移动的团子位于堆叠上方，则不会带动下方团子。
- 移动结束后，移动的整组团子落到目标点位的堆叠顶部。
- 若目标点位已有普通团子，则移动来的团子组堆叠到已有团子的上方。
- 保持移动组内部原本的上下顺序不变。

举例：

某点位堆叠 bottom -> top:
[A, B, C]

若 B 移动，则 B 和 C 一起移动，A 留在原地。
移动组为 [B, C]。

------------------------------------------------------------
游戏结束条件
------------------------------------------------------------

当某个普通团子已经经过 0 号点位，并且移动到或抵达 31 号点位时，游戏立即结算。

结算规则：

1. 先抵达 31 号点位的普通团子获得第一名。
2. 如果抵达 31 号点位时存在堆叠，则按照堆叠顺序从上到下排列名次。
3. 其余普通团子的名次按照当前位置和堆叠顺序决定。
4. 普通团子移动方向为正向，因此越接近终点 31 的团子排名越靠前。
5. 同一格内，越靠上的团子排名越靠前。
6. 哈基布不参与最终排名。

重要：

- 为避免一开始位于 0 号点位导致误判，需要记录普通团子是否已经离开并经过起点，或记录其完成进度。
- 推荐为每个普通团子维护累计正向移动进度 progress。
- 当某普通团子的 progress >= 31 时，可认为其抵达终点并触发结算。
- 地图环形位置可由 position = progress % 32 或等效逻辑计算。
- 但需要注意终点是 31，不是完整绕回 0。

------------------------------------------------------------
特殊团子：哈基布
------------------------------------------------------------

哈基布规则：

1. 游戏开始时，哈基布位于 31 号点位。
2. 哈基布不参与最终排名结算。
3. 地图所有特殊点位效果对哈基布生效。
4. 从游戏开始后的第 3 回合之后，哈基布加入随机行动序列。
   - 可以理解为：第 4 回合开始时加入行动序列。
   - 如果有歧义，请在代码注释中注明采用该解释。
5. 哈基布沿逆向方向移动，即：

31 -> 30 -> 29 -> ... -> 0 -> 31

6. 哈基布每次行动投掷 1~6 点，等概率。
7. 若哈基布在单次移动终点遇到正向行进的普通团子，则哈基布永远堆叠于最底部。
8. 哈基布不会改变普通团子的上下顺序。
9. 若在整轮行动中，任意正向前进的所有普通团子点位大于哈基布所在点位，则哈基布会传送回 31 号点位重新开始。
10. 哈基布传送时应从原点位移除，并放置到 31 号点位底部或单独位置。
11. 哈基布不参与黑色点位的普通团子随机重排。
12. 哈基布上方堆叠团子时，上方所有团子会跟随哈基布移动

建议实现方式：

- 哈基布可独立记录位置。
- 如果与普通团子同格，则视为在该格所有普通团子的最底部。
- 普通团子移动时，如哈基布在该格最底部，不会被普通团子带走，除非规则中明确需要；这里默认普通团子不会携带哈基布。
- 哈基布移动到普通团子所在格时，哈基布放在最底部，普通团子堆叠顺序保持不变。
- 每轮结束或每个普通团子行动后，检查是否满足“普通团子点位大于哈基布点位”，如果满足则哈基布传送回 31。
- 如果该规则存在歧义，请在代码注释中说明采用的检查时机。

------------------------------------------------------------
奥古斯塔
------------------------------------------------------------

技能描述：

回合开始时，若该团子处于堆叠最顶端，则在本回合不行动，且在下回合中最后一个行动。

实现要求：

- 在每回合开始时检查奥古斯塔是否处于其所在格子的最顶端。
- 如果是：
  - 奥古斯塔本回合跳过行动。
  - 下一回合奥古斯塔必须排在行动顺序最后。
- 如果下一回合存在多个强制顺序规则，请在代码中注明处理优先级。

------------------------------------------------------------
尤诺
------------------------------------------------------------

技能描述：

每场比赛一次，当该团子经过赛程中点后，若该团子排名前后有其他非哈基布团子，则将这些团子传送至自己的格子。
这些团子的堆叠顺序将与传送前的排名顺序一致。

实现要求：

- 每场比赛仅触发一次。
- “赛程中点”建议理解为进度达到或超过终点距离的一半。
- 终点距离为 31，因此中点可设为 progress >= 15.5，实现时可用 progress >= 16。
- 当尤诺首次经过中点后触发。
- 需要根据当前排名找到尤诺排名前一名和后一名的普通团子。
- 若存在这些团子，则将它们传送到尤诺当前所在格。
- 被传送团子的堆叠顺序按照传送前的排名顺序排列。
- 传送到尤诺所在格时，建议将这些团子放在该格堆叠顶部，并按照排名顺序保持从高排名到低排名的相对顺序。
- 若该规则存在歧义，请在注释中明确采用的堆叠方式。

------------------------------------------------------------
弗诺诺
------------------------------------------------------------

技能描述：

回合开始时，若该团子处于堆叠最底层，则在移动时额外前进 3 格。

实现要求：

- 每回合开始时判断弗诺诺是否位于其所在格子的最底层。
- 如果是，则本回合弗诺诺移动时额外前进 3 格。
- 普通骰子仍为 1~3，因此总移动距离为 骰子点数 + 3。
- 如果弗诺诺本回合没有行动，则加成不消耗或无效，需在代码注释中说明。

------------------------------------------------------------
长离
------------------------------------------------------------

技能描述：

如果下方堆叠其他团子，下一个回合有 65% 的概率最后一个行动。

实现要求：

- 在回合开始或回合结束时检查长离下方是否有其他普通团子。
- 如果长离下方存在其他普通团子，则下一回合有 65% 概率被安排为最后一个行动。
- 如果触发，则长离在下一回合行动顺序最后。
- 若与奥古斯塔的“最后一个行动”冲突，需要定义优先级。
- 建议：若多个团子均被指定最后行动，则它们在行动序列末尾随机排序，或按规则触发顺序排序，并在注释中说明。

------------------------------------------------------------
今汐
------------------------------------------------------------

技能描述：

如果头顶堆叠其他团子，有 40% 概率移动到所有团子的正上方。

实现要求：

- 当今汐头顶存在其他普通团子时，有 40% 概率触发。
- “移动到所有团子的正上方”理解为：今汐从当前堆叠位置移除，然后放到所在格子的最顶部。
- 该技能触发时机存在歧义。
- 建议在每次今汐行动开始前判定。
- 如果触发，则先调整堆叠，再执行移动。
- 若采用其他时机，请在代码注释中说明。

------------------------------------------------------------
卡卡罗
------------------------------------------------------------

技能描述：

开始移动时，如果在最后一名，额外前进 3 格。

实现要求：

- 在卡卡罗开始移动前判断当前排名。
- 如果卡卡罗当前排名为最后一名，则本次移动额外前进 3 格。
- 普通骰子仍为 1~3，因此总移动距离为 骰子点数 + 3。

------------------------------------------------------------
排名规则
------------------------------------------------------------

用于技能判断与最终结算。

普通团子的排名应只考虑普通团子，不包括哈基布。

排名优先级：

1. 普通团子的正向累计进度越高，排名越靠前。
2. 如果处于同一格或相同进度，则堆叠越靠上的团子排名越靠前。
3. 哈基布不计入排名。

建议维护：

progress[dango_name]
position[dango_name]
stack[position] = bottom -> top list

如果使用 progress 计算排名，需要确保因为绿色 / 红色格导致的额外移动也会同步更新 progress。

模拟输出
------------------------------------------------------------

请输出至少以下统计结果：

团子名
第一名次数
第一名概率
99% 置信区间下界
99% 置信区间上界
平均名次
各名次概率分布

99% 置信区间可使用二项分布正态近似：

p_hat = wins / n
SE = sqrt(p_hat * (1 - p_hat) / n)
CI_99 = p_hat ± 2.5758293035489004 * SE

其中：

n = 总模拟次数

如果使用更精确的 Wilson 区间，也可以，但请在输出中说明。

歧义处理建议
------------------------------------------------------------

请特别注意以下规则存在歧义，需要在实现时明确假设：

1. “游戏开始第三回合后，哈基布加入随机投掷序列”
   - 建议解释为第 4 回合开始加入。
2. 哈基布遇到普通团子后是否参与普通团子的堆叠移动
   - 建议解释为哈基布永远在最底部，但不会被普通团子带动。
3. 绿色 / 红色点位触发后，如果再次落到特殊点位，是否连锁触发
   - 建议默认只触发一次，除非规则明确连锁。
4. 尤诺技能中“排名前后”的定义
   - 建议理解为当前排名中紧邻尤诺的前一名和后一名。
5. 尤诺传送来的团子放置在尤诺所在格的具体堆叠位置
   - 建议放在该格顶部，并保持传送前排名顺序。
6. 多个团子被指定为下一回合最后行动时的冲突处理
   - 建议这些团子都放在行动序列末尾，并在它们之间随机排序。
7. 今汐技能触发时机
   - 建议在今汐行动开始前判定。
8. 长离技能触发时机
   - 建议在每回合结束时判定是否影响下一回合。

请在代码注释和 README 中明确这些假设。
