# Claude 编程指南：团子赛跑模拟器

> 本文档用于指导 Claude 使用 Python 编写一个蒙特卡洛模拟程序，根据团子赛跑游戏规则，计算各普通团子的获胜概率、平均名次、Top2 / Top3 概率，以及 99% 置信区间。

---

## 1. 任务目标

请使用 **Python** 编写一个高性能蒙特卡洛模拟程序，模拟团子赛跑游戏。

程序需要完成以下目标：

1. 至少模拟 `1,000,000` 场比赛。
2. 计算每个普通团子的：
   - 第一名胜率；
   - 平均名次；
   - Top2 概率；
   - Top3 概率；
   - 第一名胜率的 99% 置信区间。
3. 输出按胜率降序排列的统计结果表。
4. 对规则中的歧义部分，在代码注释中明确采用的解释。
5. 尽量优化性能，避免不必要的深拷贝。

---

## 2. 游戏地图

游戏地图共有 `32` 个点位，编号为：

```text
0, 1, 2, ..., 31
```

地图呈环形排列。

普通团子从 `0` 号点开始，沿正向移动：

```text
0 -> 1 -> 2 -> ... -> 31
```

当普通团子抵达 `31` 号点时，游戏进入结算。

## 3. 特殊点位

### 3.1 绿色点位

```python
GREEN_CELLS = {3, 9, 19}
```

效果：

- 当某个团子的移动终点为绿色点位时；
- 该格上的所有团子整体向前移动 1 格；
- 包括堆叠与被堆叠的团子。

### 3.2 红色点位

```python
RED_CELLS = {15, 25, 29}
```

效果：

- 当某个团子的移动终点为红色点位时；
- 该格上的所有团子整体向后移动 1 格；
- 包括堆叠与被堆叠的团子。

---

### 3.3 黑色点位

```python
BLACK_CELLS = {5, 13, 22}
```

效果：

- 当某个团子的移动终点为黑色点位时；
- 随机重新堆叠该点位上的所有普通团子；
- 不包括特殊团子“哈基布”。

## 4. 普通团子

普通团子包括：

```python
DUMPLINGS = [
    "奥古斯塔",
    "尤诺",
    "今汐",
    "卡卡罗",
    "绯雪",
    "卡提希娅",
]
```

这些团子参与最终排名与胜率统计。

## 5. 特殊团子：哈基布

特殊团子名称：

```python
SPECIAL = "哈基布"
```

哈基布规则如下：

1. 游戏开始时，哈基布位于 `31` 号点。
2. 哈基布不参与最终排名结算。
3. 地图上的所有特殊格效果对哈基布生效。
4. 游戏开始第 3 回合后，哈基布加入随机投掷序列。
5. 哈基布按照反向路线移动：

```text
31 -> 30 -> 29 -> ... -> 0
```

6. 哈基布每次行动投掷 `1-6` 点。
7. 若哈基布单次移动终点遇到正向行进的普通团子，则哈基布永远堆叠于最底部。
8. 若在整轮行动中，正向前进的普通团子点位大于哈基布所在点位，则哈基布传送回 `31` 号位重新开始。

## 6. 堆叠规则

### 6.1 初始堆叠

游戏开始时，所有普通团子以随机顺序堆叠于 `0` 号点。

越晚堆叠的团子越靠上。

推荐使用列表表示某一格的堆叠状态：

```python
board[position] = ["底部团子", ..., "顶部团子"]
```

示例：

```python
board[0] = ["奥古斯塔", "尤诺", "今汐"]
```

表示：

- `奥古斯塔` 位于底部；
- `今汐` 位于顶部。

---

### 6.2 移动时的堆叠带动

当某个普通团子行动时：

- 它会带动自己以及自己上方的所有团子一起移动；
- 它下方的团子留在原地；
- 如果它位于最顶部，则只移动自己。

示例：

```python
board[10] = ["A", "B", "C", "D"]
```

如果 `B` 行动，则移动：

```python
["B", "C", "D"]
```

`A` 留在原地。

---

### 6.3 到达目标格后的堆叠

移动团子到达目标格后，应放置到目标格已有堆叠的最上方。

示例：

```python
board[7] = ["X", "Y"]
moving_stack = ["B", "C", "D"]
```

移动后：

```python
board[7] = ["X", "Y", "B", "C", "D"]
```

---

## 7. 回合流程

每一回合按照以下流程执行：

1. 回合开始。
2. 随机排列普通团子的行动顺序。
3. 从第 3 回合后，将哈基布加入行动序列。
4. 根据行动顺序依次行动。
5. 每个普通团子在行动前后根据自身技能判断是否触发。
6. 普通团子投掷 `1-3` 点并正向移动。
7. 哈基布投掷 `1-6` 点并反向移动。
8. 处理落点特殊格效果。
9. 检查是否有普通团子抵达 `31` 号点。
10. 若有团子抵达终点，则立即结算。
11. 若无人抵达终点，则进入下一回合。

---

## 8. 游戏结束与排名

### 8.1 游戏结束条件

当任意普通团子移动至 `31` 号点时，游戏立即结算。

实现时可以简化为：

```python
if position >= 31:
    finish_game()
```

> 注意：如果代码使用环形棋盘表示普通团子位置，需要额外记录普通团子是否已经完成赛程，不能简单用取模后的位置判断终点。推荐对普通团子使用线性赛程位置，特殊格判断时再取 `position % 32`。

---

### 8.2 终点堆叠排名

若多个普通团子以堆叠形式同时抵达 `31` 号点，则按照堆叠顺序从上到下排名。

示例：

```python
board[31] = ["A", "B", "C"]
```

其中：

- `A` 在底部；
- `C` 在顶部。

则结算顺序为：

```python
["C", "B", "A"]
```

---

### 8.3 未到达终点团子的排名

对于未到达终点的团子，建议按照以下规则排序：

1. 位置越大，排名越高；
2. 若位置相同，则堆叠越靠上，排名越高。

## 9. 各团子技能

### 9.1 奥古斯塔

技能描述：

> 回合开始时，若奥古斯塔处于堆叠最顶端，则本回合不行动，且在下回合中最后一个行动。

建议维护状态：

```python
augusta_force_last_next_round = False
augusta_skip_this_round = False
```

实现逻辑：

1. 回合开始时检查奥古斯塔是否位于所在堆叠的最顶端。
2. 如果是：
   - 本回合跳过行动；
   - 下一回合最后一个行动。
3. 若需要将奥古斯塔放到最后，应在生成行动序列时处理。

推荐解释：

```text
奥古斯塔的“最后行动”指在整个行动序列中最后行动。
```

如果本回合触发“不行动”，则轮到奥古斯塔时直接跳过。

---

### 9.2 尤诺

技能描述：

> 每场比赛一次，当尤诺经过赛程中点后，若其排名前后有其他非哈基布团子，则将这些团子传送至尤诺所在格。这些团子的堆叠顺序与传送前排名顺序一致。

建议定义赛程中点：

```python
MIDPOINT = 16
```

维护状态：

```python
yuno_skill_used = False
```

触发条件：

```python
if not yuno_skill_used and position["尤诺"] >= MIDPOINT:
    trigger_yuno_skill()
```

技能效果：

1. 计算当前所有普通团子的排名。
2. 找到尤诺当前名次。
3. 找到尤诺前一名和后一名的普通团子。
4. 将这些团子从原位置移除。
5. 传送至尤诺所在格。
6. 按照传送前排名顺序重新堆叠。

推荐解释：

```text
被传送团子按照原排名从高到低，依次堆到尤诺所在格最上方。
```

---

### 9.3 今汐

技能描述：

> 如果今汐头顶堆叠其他团子，有 40% 概率移动到所有团子的正上方。

触发条件：

```python
if has_any_dumpling_above("今汐"):
    ...
```

效果：

- 若触发成功，则只移动今汐本体；
- 将今汐移动到当前格堆叠的最顶部；
- 不带动原本在今汐上方的团子。

示例：

```python
board[pos] = ["A", "今汐", "B", "C"]
```

触发后：

```python
board[pos] = ["A", "B", "C", "今汐"]
```

---

### 9.4 卡卡罗

技能描述：

> 开始移动时，如果卡卡罗处于最后一名，则额外前进 3 格。

触发时机：

```text
开始移动时
```

实现逻辑：

```python
move_steps = rng.randint(1, 3)

if rank_of("卡卡罗") == last_rank:
    move_steps += 3
```

因此卡卡罗可能移动：

```text
4, 5, 6
```

---

### 9.5 绯雪

技能描述：

> 当绯雪与哈基布相遇后，之后每次行动都额外前进 1 格。

维护状态：

```python
feixue_met_hakibu = False
```

触发条件：

- 绯雪移动到哈基布所在格；
- 哈基布移动到绯雪所在格；
- 绯雪与哈基布在同一格。

一旦触发：

```python
feixue_met_hakibu = True
```

之后绯雪每次行动：

```python
move_steps += 1
```

---

### 9.6 卡提希娅

技能描述：

> 每场比赛最多触发一次，当自身移动结束后，若处于最后一名，本场剩余回合都将有 60% 的概率前进两格。

建议维护状态：

```python
cartethyia_skill_used = False
cartethyia_skill_unlocked = False
```

触发时机：

```text
卡提希娅自身移动结束后
```

触发逻辑：

```python
if not cartethyia_skill_used and rank_of("卡提希娅") == last_rank:
    cartethyia_skill_used = True
    cartethyia_skill_unlocked = True
```

之后每次卡提希娅行动时：

```python
if cartethyia_skill_unlocked and rng.random() < 0.6:
    move_steps += 2
```

推荐解释：

```text
“前进两格”理解为在基础骰子结果之外额外 +2。
```

## 11. 哈基布移动实现建议

哈基布可以单独使用变量维护位置：

```python
hakibu_position = 31
```

哈基布按照反方向移动：

```python
hakibu_position = (hakibu_position - dice) % 32
```

当哈基布移动终点存在普通团子：

- 视为哈基布处于该格最底部；
- 但哈基布不参与普通团子的最终排名；
- 普通团子的相对堆叠顺序不因哈基布改变。

每轮结束时检查：

```python
if any(position[d] > hakibu_position for d in DUMPLINGS):
    hakibu_position = 31
```

推荐解释：

```text
“整轮行动中普通团子点位大于哈基布所在点位”在实现中按每轮结束时检查一次处理。
```

## 14. 99% 置信区间

对胜率可使用二项分布的正态近似。

```python
from math import sqrt

z = 2.5758293035489004

p = win_rate
se = sqrt(p * (1 - p) / N)

lower = max(0.0, p - z * se)
upper = min(1.0, p + z * se)
```

如果希望更稳健，也可以实现 Wilson 置信区间。

## 15. 性能优化建议

由于需要至少模拟百万场比赛，建议注意性能：

1. 使用 `random.Random(seed)` 或 `numpy.random.Generator`。
2. 棋盘只有 32 格，建议使用列表：

```python
board = [[] for _ in range(32)]
```

3. 使用 `positions` 字典维护每个团子当前位置，避免频繁全局搜索。
4. 避免在单局模拟中频繁 `deepcopy`。
5. 尽量复用小型列表和局部变量。
6. 如需进一步加速，可使用：
   - `multiprocessing`
   - `numba`
   - 多进程分批模拟后汇总结果。

## 18. 建议补充说明

由于原规则中存在一些可能产生歧义的地方，建议在代码中明确以下解释：

1. **奥古斯塔的“最后行动”**  
   推荐解释为：在整个行动序列中最后行动，包括哈基布之后。

2. **尤诺传送团子的堆叠方式**  
   推荐解释为：按照传送前排名从高到低，依次放到尤诺所在格最上方。

3. **哈基布重置时机**  
   推荐解释为：每轮结束时检查是否存在普通团子位置大于哈基布位置，若存在则哈基布回到 31 号点。

4. **卡提希娅的额外前进两格**  
   推荐解释为：在基础骰子结果之外额外增加 2 格。

5. **绿色 / 红色格触发后的连锁效果**  
   建议明确是否允许连续触发特殊格。推荐实现为：一次移动结束后只处理当前落点的特殊格效果一次，避免无限连锁。

## 22. 最终提醒

本模拟属于随机蒙特卡洛方法。即使规则完全相同，不同随机种子也可能导致统计结果存在细微差异。

因此，输出结果时建议同时给出：

- 模拟次数；
- 随机种子；
- 置信区间；
- 对歧义规则的解释；
- 代码版本或提交哈希。
